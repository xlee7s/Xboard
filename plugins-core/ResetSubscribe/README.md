# 重置订阅-订阅自焚插件（reset_subscribe）

## 📌 插件简介

**重置订阅-订阅自焚插件**用于在 Xboard 系统中定时重置用户的`token`，从而使原订阅链接失效并重新生成，实现“订阅自焚 + 自动重置”的安全机制。

适用于需要：
- 定期轮换订阅链接
- 防止订阅链接长期泄露
- 强制用户刷新配置
- 提升订阅安全性的场景

---

## 🚀 插件特性

- ⏱ 支持按分钟级定时执行
- 🔁 自动重置用户token
- 💣 实现订阅链接“自焚”机制
- 🛠️ 可选运行日志记录
- 🔧 可后台开关控制任务启停
- 🛡 最低执行间隔强制限制（≥1分钟）

---

## ⚙️ 配置说明

### interval_minutes（重置间隔 / 分钟）
- 类型：number  
- 默认值：5  
- 说明：每隔多少分钟执行一次订阅重置任务（最小 1 分钟）

---

### enable_log（开启日志）
- 类型：boolean  
- 默认值：true  
- 说明：是否记录插件运行日志，便于排查问题

---

### status（是否开启任务）
- 类型：boolean  
- 默认值：false  
- 说明：控制是否启用自动重置订阅任务

---

## 🧠 工作机制

1. 系统按 `interval_minutes` 进入调度周期  
2. 任务触发后扫描目标用户列表  
3. 为每个用户生成新的：
   - token  
4. 旧订阅链接立即失效（“自焚”效果）  
5. 新订阅数据写入数据库  
6. 可选记录执行日志

---

## 📖 使用方法

### 1️⃣ 安装插件

将插件放入 Xboard 插件目录：`plugins`


然后在后台刷新插件列表或执行安装操作。

---

### 2️⃣ 启用插件

进入后台插件管理：

- 找到「重置订阅-订阅自焚」并启用

---

### 3️⃣ 配置执行频率

设置 `interval_minutes`：

- 推荐：5 ~ 60 分钟
- 最低：1 分钟（系统强制限制）

---

### 4️⃣ 开启日志（可选）

将 `enable_log` 设置为 `true`：

- 用于记录每次重置执行情况
- 便于排查用户订阅异常问题
